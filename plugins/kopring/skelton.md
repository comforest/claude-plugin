# 핵심 목표
1. 정해진 워크플로우에 맞춰 작업 진행
2. 토큰 소모 최적화

# 지켜야 할 원칙
1. 작업이 중도 중지되어도 새로운 세션에서 재개될 수 있도록 기록이 필요. 가급적 md 파일로 기록
2. 언제나 리뷰는 새로운 세션에서 진행 (컨텍스트 전달이 없는 subAgent 에서 진행)
3. skill 당 권장 라인 150Line, 최대 200 Line
4. 병렬을 활용할 필요는 없으나, subagent는 적극 활용한다.
   1. SubAgent는 작업에 맞춰 원하는 모델을 선택하여 토큰 낭비를 줄인다.
   2. 컨텍스트를 나눠 작업하여 토큰 낭비를 줄인다.

# 워크 플로우 (메인 4단계, 자동 체이닝)
`startPlan` 한 번 호출로 4단계가 자동으로 이어진다. 자동 체인이 멈추는 지점은 두 곳뿐:
- `review`(plan 모드)에서 Critical/Major 발견 → 사용자가 `startPlan` 재호출로 본문 갱신
- 색션 PR 생성 후 → 머지가 외부 이벤트라 다음 색션은 사용자가 `execute` 재호출

1. 요구 사항을 명확히 하여 작업 계획을 세운다. (`startPlan` 스킬) — 작성 완료 시 자동으로 `review` 호출.
    1. 작업 계획을 나눌때는 작업 크기에 따라 색션을 나눈다. 이 색션은 추후 PR 단위가 된다.
2. 작업 계획을 리뷰한다. (`review` 스킬, 항상 새 세션 subagent에서) — 통과 시 자동으로 `execute` 호출.
3. 작업 계획대로 진행한다. (`execute` 스킬, 한 번에 한 색션) — 색션 완료 시 자동으로 `pr` 호출.
4. PR을 작성한다. (`pr` 스킬) — PR 생성 후 자동 체인 종료.
    1. PR 본문에는 지라 링크와 색션 작업 항목이 포함된다.
    2. 색션이 여러 개면 sub-branch(`feat/PROJ-1-N`)로 stacked PR.

# 보조 스킬
- `pr-review` — origin PR에 달린 리뷰 코멘트를 그룹 단위 커밋으로 반영
- `pr-upstream` — fork → upstream PR 생성 (origin PR 본문 재활용)
